## 🔧 修正方法

**ファイルマネージャーからテーマファイル（header.php）を修正**

### 実施内容

1. **AIOSEOプラグインでの設定を試行**
   - 各ページ（`/contact/`, `/career/`, `/entry/`）のAIOSEO設定でタイトルとメタディスクリプションを設定
   - しかし、HTMLソースコードに `<title>` タグが出力されない問題が発生

2. **原因の特定**
   - テーマの `header.php` に `<title>` タグが存在しない
   - AIOSEOが他のメタタグ（description、og:title等）は出力しているが、`<title>` タグだけが出力されない

3. **解決方法**
   - ファイルマネージャーから `header.php` をダウンロード
   - `<?php wp_head(); ?>` の後に、条件分岐でタイトルタグを追加
   - 修正したファイルをアップロード
   - これにより、確実にHTMLにタイトルタグが出力されることを確認

### 修正コードの追加箇所

ファイルマネージャーで `header.php` を編集し、`<?php wp_head(); ?>` の直後に以下を追加：

```php
<?php
// SEO設定：タイトルタグを強制的に出力（AIOSEOの後に追加）
if (is_page('career')) : ?>
    <title>採用情報・求人｜プラントエンジニア募集中｜コモンウェルスエンジニアーズ</title>
<?php elseif (is_page('contact')) : ?>
    <title>お問い合わせ・ご相談｜プラント設計のコモンウェルスエンジニアーズ</title>
<?php elseif (is_page('entry')) : ?>
    <title>採用エントリーフォーム｜中途・新卒採用｜コモンウェルスエンジニアーズ</title>
<?php endif; ?>
```

## 📝 その他気づいた点

### 1. AIOSEOプラグインの動作について

- AIOSEOプラグインは、メタディスクリプション、Open Graphタグ、Twitter Cardタグなどは正常に出力している
- しかし、`<title>` タグだけが出力されない問題が発生
- 原因は、テーマの `header.php` に `<title>` タグが存在しないため、AIOSEOがタイトルを出力するフックが機能していない可能性

### 2. ファイルマネージャーからの修正について

- WordPress管理画面のテーマファイルエディターではなく、ファイルマネージャー（FTP/サーバーのファイルマネージャー）から修正を実施
- これにより、コードの崩れを防ぎ、安全に修正できた
- 子テーマを使用していない場合、テーマ更新時に上書きされる可能性がある
- 将来的には、子テーマを作成してから編集することを推奨

### 3. Search Consoleでの確認

- URL検査で各ページを確認
- 「インデックス登録をリクエスト」ボタンが表示されない場合もあり（既にインデックス済み、または自動クロールを待つ状態）
- タイトルタグがHTMLに正しく出力されていれば、Googleの自動クロールで検出されるため、問題なし

### 4. 今後の確認事項

- **1週間後**: 検索結果でタイトルが表示されているか確認
- **2週間後**: Search Consoleの「検索パフォーマンス」でCTR（クリック率）の変化を確認
