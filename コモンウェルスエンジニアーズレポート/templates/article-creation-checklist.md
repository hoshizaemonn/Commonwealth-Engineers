# 記事作成チェックリスト

このチェックリストを使用して、記事作成時の品質管理を行ってください。

---

## 📋 記事作成ワークフロー

### Phase 1: 記事作成前の準備

#### 1.1 客先コメントの確認

- [ ] `articles/客先コメント/` フォルダーを確認
- [ ] 最新のコメントPDFを確認
- [ ] 過去のコメントで指摘された表現をリストアップ

#### 1.2 参照資料の確認

- [ ] 類似記事を確認（過去記事フォルダー）
- [ ] 記事作成ガイドライン（`knowledge/article-writing-guidelines.md`）を確認
- [ ] 禁止表現リストを確認

---

### Phase 2: 記事作成中

#### 2.1 表現チェック

- [ ] ❌ 「プロの設計者」「プロの○○」という表現を使用していない
- [ ] ✅ 「実務」「実践的」「実際の業務」などの適切な代替表現を使用
- [ ] ❌ 「成功裏に完了」などの断定表現を使用していない
- [ ] ✅ 「携わらせていただく」などの謙虚な表現を使用
- [ ] ❌ 専門用語（「セッション数」など）を多用していない
- [ ] ✅ 一般の方にも分かりやすい表現を使用
- [ ] 表現が謙虚で、上から目線になっていない

#### 2.2 SEO設定

- [ ] Meta Title を設定（30-60文字程度）
- [ ] Meta Description を設定（120-160文字程度）
- [ ] Keywords を設定（5-10個程度）
- [ ] 重要なキーワードを本文に適切に配置

#### 2.3 構成・見出し

- [ ] 見出し構造（H2, H3, H4）が適切
- [ ] 見出しが内容を正確に表している
- [ ] 表やリストで情報を整理している
- [ ] 重要な部分を`<strong>`タグで強調

#### 2.4 技術的要素

- [ ] Markdown版（`.md`）を作成
- [ ] WordPress用HTML版（`_wp.html`）を作成
- [ ] フォントサイズが適切（本文19px以上）
- [ ] 行間が適切（読みやすさを考慮）
- [ ] スタイルが統一されている

---

### Phase 3: 完成後のチェック

#### 3.1 最終確認

- [ ] 禁止表現リストと照らし合わせて全文を確認
- [ ] 客先コメントで指摘されたポイントが反映されている
- [ ] リンターエラーがない
- [ ] スペルチェック・誤字脱字チェック
- [ ] Markdown版とHTML版の内容が一致している

#### 3.2 品質チェック

- [ ] 文章が自然で読みやすい
- [ ] 情報が正確である
- [ ] 数値や日付が正しい
- [ ] リンクが正しく設定されている
- [ ] 画像のパスが正しい（使用している場合）

---

### Phase 4: 提出準備

#### 4.1 ファイル準備

- [ ] Markdown版（`.md`）を適切なフォルダーに保存
- [ ] WordPress用HTML版（`_wp.html`）を適切なフォルダーに保存
- [ ] ファイル名が規則に従っている（例：`YYYYMMDD_記事名.md`）

#### 4.2 メール作成

- [ ] 記事案提出メールテンプレートを使用
- [ ] 記事の概要を記載
- [ ] SEO設定を記載
- [ ] 添付ファイルを確認

---

## 🚫 禁止表現クイックチェック

記事内で以下の表現が含まれていないか確認：

- [ ] 「プロの設計者」
- [ ] 「プロの○○」（○○には設計環境、目、背中、世界など）
- [ ] 「成功裏に完了」
- [ ] 「セッション数」（専門用語）
- [ ] その他、客先コメントで指摘された表現

見つかった場合は、**必ず**適切な代替表現に修正してください。
